Lesson 6 - Part 4

S3 Bucket for Data Upload

Part 4: S3 Bucket for Data Upload (30 minutes)

Step 10: Create S3 Bucket

S3STS
# Create bucket
export BUCKET_NAME="data-pipeline-uploads-$(aws sts get-caller-identity --query Account --output text)"

aws s3 mb s3://$BUCKET_NAME

echo "S3 Bucket: $BUCKET_NAME"

Step 11: Create Sample CSV Data

Create sample-sales.csv:

date,product,quantity,revenue,region
2024-01-01,Widget A,100,1500.00,North
2024-01-01,Widget B,75,1125.50,South
2024-01-01,Gadget X,50,2500.00,East
2024-01-02,Widget A,120,1800.00,North
2024-01-02,Gadget Y,30,900.00,West
2024-01-03,Widget B,200,3000.00,South

What's happening here?

  • Created an S3 bucket to receive data uploads
  • Prepared sample CSV data to test the pipeline
  • This bucket will be the trigger for our data processing

The event-driven pattern:

  1. User/System uploads CSV to S3
  2. S3 sends event notification
  3. Lambda automatically processes the file
  4. Data lands in RDS, cache gets invalidated
  5. No servers to manage, scales automatically!

Pro tip: S3 event notifications are the foundation of serverless data pipelines. They're instant, reliable, and free!
