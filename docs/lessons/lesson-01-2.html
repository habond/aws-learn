What Makes a Good Static Site for This Lesson:

  • HTML/CSS/JavaScript only (no server-side code)
  • Images (so we can test CDN performance)
  • Multiple pages (to test routing)
  • Some CSS/JS files (to test caching)
üìò What's the difference between Static and Dynamic websites?

What it is: A static website is pre-built HTML/CSS/JS files served exactly as stored - same content for everyone. A dynamic website generates custom pages on-the-fly based on who's viewing, using databases and server code (like PHP, Python, Node.js).

Why we need it: Static sites are perfect for content that doesn't change per user: portfolios, documentation, landing pages, blogs (with build tools). They're 10-100x cheaper to host because you don't need servers - just storage. They're also faster and more secure (no server to hack).

Key details: Modern static sites aren't boring - you can use JavaScript to make them interactive. Many blogs use "static site generators" like Jekyll or Hugo that build static HTML from templates, getting benefits of both worlds. You update by rebuilding and re-uploading.

Common gotcha: Users can't log in, submit forms, or see personalized content without adding external services (like Auth0 for login, FormSpree for forms). If you need "admin updates blog without rebuilding", you need either a dynamic site or a headless CMS with a static site generator.
